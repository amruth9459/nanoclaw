# Google Workspace Integration

OAuth-based integration giving the NanoClaw container agent access to Gmail, Calendar, Drive, and Keep.

## Prerequisites

1. **Google Cloud Project** with these APIs enabled:
   - Gmail API
   - Google Calendar API
   - Google Drive API
   - Google Keep API

2. **OAuth 2.0 Client ID** (Desktop application type):
   - Google Cloud Console → APIs & Services → Credentials
   - Create Credentials → OAuth client ID → Desktop app
   - Download the JSON and save as `~/.config/gws/credentials.json`

3. **Python packages:**
   ```bash
   pip install google-auth google-auth-oauthlib google-api-python-client
   ```

## Setup

```bash
# First time — opens browser for consent
python3 gws/setup_oauth.py

# With API tests
python3 gws/setup_oauth.py --test

# Check current token status
python3 gws/setup_oauth.py --check

# Refresh expired token (no browser)
python3 gws/setup_oauth.py --refresh-only
```

The OAuth flow will request these scopes (least privilege):

| Scope | Access Level | Purpose |
|-------|-------------|---------|
| `gmail.modify` | Read + send + trash + archive | Email tools (no permanent delete) |
| `calendar.readonly` | Read-only | Calendar events |
| `drive.readonly` | Read-only | File search |
| `keep.readonly` | Read-only | Notes (Phase 1) |

## Architecture

```
Host: ~/.config/gws/
├── credentials.json    (OAuth client, from Google Cloud Console)
└── token.json          (OAuth token with refresh_token, generated by setup)

Container mount (read-only):
  ~/.config/gws/ → /workspace/gws/tokens/

Container code:
  /workspace/gws/gws_helper.py  (Python helper, shells out from gws-tools.ts)
```

`container-runner.ts` mounts `~/.config/gws/` read-only into containers. The Python helper refreshes tokens in-memory when expired; the refreshed token can't persist on the read-only mount but works for the session.

## Security

- **Token directory**: `~/.config/gws/` with mode `0700`
- **Token files**: mode `0600`
- **Git**: `gws/tokens/` and `**/tokens/token.json` are in `.gitignore`
- **Container mount**: read-only — agent cannot modify token files
- **Error handling**: `gws_helper.py` sanitizes error messages to prevent `client_secret` leakage
- **No permanent delete**: `gmail.modify` allows trash (recoverable) but not permanent deletion

## Troubleshooting

**"GWS helper not found"** — Container doesn't have the gws skill. Check that `gws-tools` is registered for the group.

**"OAuth token expired and refresh failed"** — Re-run `python3 gws/setup_oauth.py` on the host to get a new token.

**"Missing scopes"** after `--check` — Token was created with fewer scopes. Re-run full auth flow (without `--refresh-only`) to get all scopes.

**Keep API 403** — Google Keep API may need to be enabled separately in Google Cloud Console → APIs & Services → Library → search "Keep".
